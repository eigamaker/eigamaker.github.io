1. Resumen

Este Acuerdo de Licencia de Usuario Final (« Acuerdo ») es un contrato legalmente vinculante entre usted (« Usuario ») y el operador del servicio Profilecode (« Operador »). Este Acuerdo se aplica a la aplicación iOS/Android y la Web (profilecode.codes) (en conjunto, el « Servicio »).

Al acceder o utilizar el Servicio, se considera que acepta este Acuerdo. Si no está de acuerdo, no utilice el Servicio.

2. Definiciones

  • « Datos de entrada del usuario »: Información ingresada por el usuario como parte del uso del Servicio, como respuestas a preguntas de diagnóstico y nombres para mostrar opcionales
  • « Puntuación »: Valores numéricos o categóricos calculados basados en los datos de entrada del usuario
  • « Perfil »: Resultados de análisis de personalidad presentados por el Servicio basados en puntuaciones
  • « Resultado de prueba de compatibilidad »: Resultados del análisis de compatibilidad entre usuarios o grupos
  • « Cuenta »: Un estado de usuario identificado por dirección de correo electrónico, etc., que permite la sincronización de datos del lado del servidor
  • « Uso anónimo »: Uso del Servicio sin crear una cuenta (principalmente almacenado en el dispositivo)
  • « Enlace compartido »: Un enlace (token) que un usuario puede compartir a través de canales externos (redes sociales, etc.) para pruebas de compatibilidad

3. Elegibilidad (Edad)

El Servicio está dirigido a usuarios de 16 años o más en principio. Puede que no verifiquemos la edad, pero al usar el Servicio, el usuario declara que tiene al menos 16 años. Si tiene menos de 16 años, no utilice el Servicio.

7. Datos de entrada del usuario y resultados generados automáticamente

Estos resultados son generados automáticamente y no constituyen asesoramiento profesional (médico, psicoterapéutico, legal, de inversión, etc.). Cómo utiliza los resultados es su responsabilidad, y el Operador no garantiza la exactitud, integridad o idoneidad para un propósito particular.

El Servicio no proporciona funciones de contenido generado por usuarios (UGC) de publicación libre como tablones de anuncios o comentarios. Sin embargo, pueden ocurrir entradas como nombres para mostrar e interacciones indirectas a través de enlaces compartidos (Sección 7.1).

7.1 Enlaces compartidos (compartir enlaces e interacción indirecta)

  • Los enlaces compartidos pueden ser entregados a terceros a través de canales externos (redes sociales, aplicaciones de mensajería, etc.).
  • Los enlaces compartidos pueden ser accesibles sin inicio de sesión. Como los enlaces compartidos pueden incluir información visible por los destinatarios, manéjelos con cuidado.
  • El período de validez de un enlace compartido es, en principio, de 30 días desde la emisión.
  • Los enlaces compartidos actualmente no pueden ser revocados (invalidados) en un momento arbitrario. Los usuarios deben compartir teniendo esto en cuenta.
  • Los destinatarios de un enlace compartido pueden sincronizar el ID de usuario, nombre para mostrar (si está configurado), puntuaciones, etc. del usuario compartido a su propio dispositivo para pruebas de compatibilidad. No se proporciona entrada adicional ni mensajería directa en la aplicación.
  • Cuando un usuario entrega un enlace compartido a un tercero, se considera que el usuario ha consentido proporcionar la información necesaria para ese intercambio.
  • Si se sospecha mal uso de enlaces (acoso, suplantación de identidad, uso no autorizado, etc.), el Operador puede restringir el acceso o suspender cuentas.

8. Almacenamiento de datos y privacidad

8.1 Uso anónimo (sin cuenta)

  • Web: Las respuestas de diagnóstico pueden almacenarse en el almacenamiento del navegador (por ejemplo, memoria/almacenamiento web). Pueden eliminarse borrando los datos del navegador.
  • Aplicación: Las respuestas de diagnóstico pueden almacenarse en bases de datos locales sin conexión. Pueden eliminarse eliminando la aplicación o mediante operaciones del dispositivo.

8.2 Uso de cuenta (creación/vinculación de cuenta)

  • Si un usuario crea una cuenta o se convierte de uso anónimo (por ejemplo, configuración de correo electrónico/contraseña), el Servicio puede enviar y almacenar datos en servidores (por ejemplo, Supabase) para sincronización de datos (por ejemplo, transferencia de respuestas/historial del dispositivo).
  • Información que puede almacenarse: Dirección de correo electrónico, ID de usuario, respuestas de diagnóstico, puntuaciones (brutas/agregadas), historial de respuestas, tokens de enlaces compartidos, información de grupo, etc.
  • Período de retención: En principio, hasta que el usuario elimine la cuenta

8.3 Eliminación de cuenta

Cuando un usuario elimina una cuenta, el Operador eliminará, dentro de un período razonable, los datos del usuario, los tokens de enlaces compartidos y la información relacionada con grupos en el servidor. Sin embargo, pueden aplicarse excepciones cuando la retención sea requerida para cumplimiento legal, seguridad o resolución de disputas.

Para detalles sobre el manejo de datos, servicios de terceros, publicidad/medición y derechos del usuario, consulte la Política de Privacidad.
