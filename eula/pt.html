Contrato de Licença de Usuário Final (EULA)

Nome do serviço: Profilecode

Versão: 3.0

Última atualização: 20 de janeiro de 2026

1. Visão geral

Este Contrato de Licença de Usuário Final ("Contrato") é um acordo legalmente vinculativo entre você ("Usuário") e o operador do serviço Profilecode ("Operador"). Este Contrato se aplica ao aplicativo iOS/Android e à Web (profilecode.codes) (coletivamente, o "Serviço").

Ao acessar ou usar o Serviço, você é considerado como tendo concordado com este Contrato. Se você não concordar, não use o Serviço.

2. Definições

  • "Dados de entrada do usuário": Informações inseridas pelo usuário como parte do uso do Serviço, como respostas a perguntas de diagnóstico e nomes de exibição opcionais
  • "Pontuação": Valores numéricos ou categóricos calculados com base nos dados de entrada do usuário
  • "Perfil": Resultados da análise de personalidade apresentados pelo Serviço com base em pontuações
  • "Resultado do teste de compatibilidade": Resultados da análise de compatibilidade entre usuários ou grupos
  • "Conta": Um status de usuário identificado por endereço de e-mail, etc., que permite sincronização de dados do lado do servidor
  • "Uso anônimo": Uso do Serviço sem criar uma conta (principalmente armazenado no dispositivo)
  • "Link compartilhado": Um link (token) que um usuário pode compartilhar através de canais externos (redes sociais, etc.) para testes de compatibilidade

3. Elegibilidade (Idade)

O Serviço é destinado a usuários com 16 anos ou mais em princípio. Podemos não verificar a idade, mas ao usar o Serviço, o usuário declara que tem pelo menos 16 anos. Se você tiver menos de 16 anos, não use o Serviço.

7. Dados de entrada do usuário e resultados gerados automaticamente

O Serviço calcula pontuações com base nos dados de entrada do usuário e apresenta perfis e resultados de testes de compatibilidade. Esses resultados são gerados automaticamente e não constituem aconselhamento profissional (médico, psicoterapêutico, legal, de investimento, etc.). Como você usa os resultados é de sua responsabilidade, e o Operador não garante precisão, integridade ou adequação para um propósito particular.

O Serviço não fornece recursos de conteúdo gerado pelo usuário (UGC) de publicação livre, como fóruns ou comentários. No entanto, podem ocorrer entradas como nomes de exibição e interações indiretas através de links compartilhados (Seção 7.1).

7.1 Links compartilhados (compartilhamento de links e interação indireta)

  • Links compartilhados podem ser entregues a terceiros através de canais externos (redes sociais, aplicativos de mensagens, etc.).
  • Links compartilhados podem ser acessíveis sem login. Como links compartilhados podem incluir informações visualizáveis pelos destinatários, manuseie-os com cuidado.
  • O período de validade de um link compartilhado é, em princípio, de 30 dias a partir da emissão.
  • Links compartilhados atualmente não podem ser revogados (invalidados) em um momento arbitrário. Os usuários devem compartilhar tendo isso em mente.
  • Os destinatários de um link compartilhado podem sincronizar o ID do usuário, nome de exibição (se definido), pontuações, etc. do usuário compartilhado para seu próprio dispositivo para testes de compatibilidade. Não são fornecidas entradas adicionais ou funcionalidades de mensagens diretas no aplicativo.
  • Quando um usuário entrega um link compartilhado a um terceiro, considera-se que o usuário consentiu em fornecer as informações necessárias para esse compartilhamento.
  • Se houver suspeita de uso indevido de links (assédio, impersonificação, uso não autorizado, etc.), o Operador pode restringir o acesso ou suspender contas.

8. Armazenamento de dados e privacidade

8.1 Uso anônimo (sem conta)

  • Web: As respostas de diagnóstico podem ser armazenadas no armazenamento do navegador (por exemplo, memória/armazenamento web). Podem ser removidas limpando os dados do navegador.
  • Aplicativo: As respostas de diagnóstico podem ser armazenadas em bancos de dados offline locais. Podem ser removidas excluindo o aplicativo ou através de operações do dispositivo.

8.2 Uso da conta (criação/vinculação de conta)

  • Se um usuário criar uma conta ou converter de uso anônimo (por exemplo, configuração de e-mail/senha), o Serviço pode enviar e armazenar dados em servidores (por exemplo, Supabase) para sincronização de dados (por exemplo, transferência de respostas/histórico do dispositivo).
  • Informações que podem ser armazenadas: Endereço de e-mail, ID do usuário, respostas de diagnóstico, pontuações (brutas/agregadas), histórico de respostas, tokens de links compartilhados, informações de grupo, etc.
  • Período de retenção: Em princípio, até que o usuário exclua a conta

8.3 Exclusão de conta

Quando um usuário exclui uma conta, o Operador excluirá, dentro de um período razoável, os dados do usuário, tokens de links compartilhados e informações relacionadas a grupos no servidor. No entanto, podem se aplicar exceções quando a retenção for necessária para conformidade legal, segurança ou resolução de disputas.

Para detalhes sobre o tratamento de dados, serviços de terceiros, publicidade/medição e direitos do usuário, consulte a Política de Privacidade.
