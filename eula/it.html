1. Panoramica

Questo Contratto di Licenza Utente Finale ("Contratto") è un accordo legalmente vincolante tra te ("Utente") e l'operatore del servizio Profilecode ("Operatore"). Questo Contratto si applica all'app iOS/Android e al Web (profilecode.codes) (collettivamente, il "Servizio").

Accedendo o utilizzando il Servizio, si considera che tu accetti questo Contratto. Se non sei d'accordo, non utilizzare il Servizio.

2. Definizioni

  • "Dati di input dell'utente": Informazioni inserite dall'utente nell'ambito dell'utilizzo del Servizio, come risposte alle domande diagnostiche e nomi visualizzati opzionali
  • "Punteggio": Valori numerici o categorici calcolati sulla base dei dati di input dell'utente
  • "Profilo": Risultati dell'analisi della personalità presentati dal Servizio sulla base dei punteggi
  • "Risultato del test di compatibilità": Risultati dell'analisi di compatibilità tra utenti o gruppi
  • "Account": Uno stato utente identificato da indirizzo email, ecc., che consente la sincronizzazione dei dati lato server
  • "Utilizzo anonimo": Utilizzo del Servizio senza creare un account (principalmente memorizzato sul dispositivo)
  • "Link condiviso": Un link (token) che un utente può condividere tramite canali esterni (social media, ecc.) per test di compatibilità

3. Idoneità (Età)

Il Servizio è destinato agli utenti di 16 anni o più in linea di principio. Potremmo non verificare l'età, ma utilizzando il Servizio, l'utente dichiara di avere almeno 16 anni. Se hai meno di 16 anni, non utilizzare il Servizio.

7. Dati di input dell'utente e risultati generati automaticamente

Il Servizio calcola i punteggi sulla base dei dati di input dell'utente e presenta profili e risultati di test di compatibilità. Questi risultati sono generati automaticamente e non costituiscono consulenza professionale (medica, psicoterapeutica, legale, di investimento, ecc.). Come utilizzi i risultati è di tua responsabilità, e l'Operatore non garantisce accuratezza, completezza o idoneità per uno scopo particolare.

Il Servizio non fornisce funzionalità di contenuti generati dagli utenti (UGC) a pubblicazione libera come forum o commenti. Tuttavia, possono verificarsi input come nomi visualizzati e interazioni indirette tramite link condivisi (Sezione 7.1).

7.1 Link condivisi (condivisione di link e interazione indiretta)

  • I link condivisi possono essere trasmessi a terzi tramite canali esterni (social media, app di messaggistica, ecc.).
  • I link condivisi possono essere accessibili senza accesso. Poiché i link condivisi possono includere informazioni visualizzabili dai destinatari, gestiscili con cura.
  • Il periodo di validità di un link condiviso è, in linea di principio, di 30 giorni dall'emissione.
  • I link condivisi attualmente non possono essere revocati (invalidati) in un momento arbitrario. Gli utenti dovrebbero condividere tenendo conto di questo punto.
  • I destinatari di un link condiviso possono sincronizzare l'ID utente, il nome visualizzato (se impostato), i punteggi, ecc. dell'utente condiviso sul proprio dispositivo per test di compatibilità. Non sono forniti input aggiuntivi o funzionalità di messaggistica diretta nell'app.
  • Quando un utente trasmette un link condiviso a un terzo, si considera che l'utente abbia acconsentito a fornire le informazioni necessarie per tale condivisione.
  • Se si sospetta un abuso di link (molestie, impersonificazione, utilizzo non autorizzato, ecc.), l'Operatore può limitare l'accesso o sospendere gli account.

8. Archiviazione dei dati e privacy

8.1 Utilizzo anonimo (nessun account)

  • Web: Le risposte diagnostiche possono essere memorizzate nell'archiviazione del browser (ad esempio, memoria/archiviazione web). Possono essere rimosse cancellando i dati del browser.
  • App: Le risposte diagnostiche possono essere memorizzate in database offline locali. Possono essere rimosse eliminando l'app o tramite operazioni sul dispositivo.

8.2 Utilizzo dell'account (creazione/collegamento account)

  • Se un utente crea un account o passa da utilizzo anonimo (ad esempio, configurazione email/password), il Servizio può inviare e memorizzare dati su server (ad esempio, Supabase) per la sincronizzazione dei dati (ad esempio, trasferimento di risposte/cronologia del dispositivo).
  • Informazioni che possono essere memorizzate: Indirizzo email, ID utente, risposte diagnostiche, punteggi (grezzi/aggregati), cronologia delle risposte, token di link condivisi, informazioni di gruppo, ecc.
  • Periodo di conservazione: In linea di principio, fino a quando l'utente elimina l'account

8.3 Eliminazione dell'account

Quando un utente elimina un account, l'Operatore eliminerà, entro un periodo ragionevole, i dati utente, i token di link condivisi e le informazioni relative ai gruppi sul server. Tuttavia, possono applicarsi eccezioni quando la conservazione è richiesta per conformità legale, sicurezza o risoluzione delle controversie.

Per dettagli sul trattamento dei dati, servizi di terze parti, pubblicità/misurazione e diritti degli utenti, consultare l'Informativa sulla Privacy.
